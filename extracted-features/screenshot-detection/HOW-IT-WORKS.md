# Screenshot Detection Feature — How It Works

## Overview

This is **not** a screenshot capture feature. It is a lightweight **metadata detection** mechanism that identifies whether an existing image file was originally a screenshot (as opposed to a camera photo).

## How Detection Works

The detection is based on a single heuristic: **screenshots have no EXIF exposure time**.

### `DImageMeta.kt` (shared data model)

```kotlin
data class DImageMeta(
    val exposureTime: String,  // e.g. "1/120" for camera photos, "" for screenshots
    ...
) {
    val isScreenshot: Boolean
        get() = exposureTime.isEmpty()  // ← The detection logic
}
```

EXIF data is extracted from image files using `ImageHelper.getMeta(path)`. The `exposureTime` field is populated from EXIF tag `TIFF_TAG_EXPOSURE_TIME`. Screenshots generated by Android do not contain this EXIF tag, so the field remains empty.

### Where It Is Used

`ImageMetaRows.kt` (Composable UI component):

```kotlin
meta?.let { mt ->
    if (mt.isScreenshot) {
        return  // Skip showing EXIF details for screenshots
    }
    // Show camera metadata: aperture, ISO, flash, etc.
}
```

When viewing an image's metadata, if the image is detected as a screenshot, no EXIF camera details (taken at, resolution, device make/model, aperture, exposure time, focal length, ISO, flash, f-number, exposure program, metering mode, white balance, creator, description) are displayed.
